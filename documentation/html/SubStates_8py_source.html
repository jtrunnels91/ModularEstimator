13 # @brief This is an abstract base class for objects used as sub-states in
14 # State.ModularFilter.
15 #
16 # @details SubState is an abstract base class that specifies the methods which
17 # are required for an object to function as a sub-state of State.ModularFilter.
18 #
19 # Some of these methods are implemented and most likely do not need to be
20 # reimplemented in a derived class implementation (for example the #dimension
21 # and #covariance methods.
22 #
23 # Other methods may have a rudimentary implementation that may be suitable for
24 # some derived classes, but not others, depending on the specific
25 # functionality of the derived class (for instance #getStateVector and
26 # #storeStateVector).
27 #
28 # Finally, some methods are specifically tagged as abstract methods and are
29 # not implemented at all. These methods must be implemented in the derived
30 # class. This is usually because there is no way to implement even a
31 # rudimentary version of what the method is supposed to do without having some
32 # knowledge of what kind of substate the derived class contains (for instance
33 # #timeUpdate and #getMeasurementMatrices).
34 #
35 # In any case, the documentation for each method of SubState contains a
36 # generalized description of what functionality the implementation should
37 # provide in a derived class.

184  ## @fun #timeUpdate returns time-update matrices
185  #
186  # @details The #timeUpdate method is responsible for returning the EKF
187  # time update measurement matrices. Specifically, it returns the state
188  # update matrix \f$\mathbf{F}\f$ and the process noise matrix
189  # \f$\mathbf{Q}\f$, following the standard
190  # <a href="https://en.wikipedia.org/wiki/Extended_Kalman_filter">
191  # Extended Kalman Filter</a> time update equations:
192  #
193  # \f[\sv[timeIndex=k+1, aPriori=True] =
194  # \mathbf{F} \sv[timeIndex=k, aPriori=False] \f]
195  # \f[\svVar[timeIndex=k+1, aPriori=True] =
196  # \mathbf{F} \svVar[timeIndex=k+1, aPriori=True] \mathbf{F}^{T} + \mathbf{Q} \f]
197  #
198  # Because these matrices are nescessarily specific to the type of substate
199  # being updated, there is no default implementation in the SubState class.
200  # Rather, each derived class must implement this method as appropriate for
201  # the dynamics of the state being modeled.
202  #
203  # In addition, some substates may require additional operations to occur
204  # at a time update. For instance, if a substate includes auxillary values
205  # (for instance, the attitude quaternion derived from the attitude error
206  # state), it may need to be time-updated seperately from the other states.
207  # In this case, the local implementation of the #timeUpdate function is
208  # the place to do these updates.
209  #
210  # @param self The object pointer
211  # @param dT The ellapsed time over which the time update occurs
212  # @param dynamics A dictionary containing any dynamics infomation which
213  # may be needed to update the state, for instance, measured accelerations
214  # or angular velocities.
215  #
216  # @returns A dictionary containing, at minimum, the following items:
217  # - "F": The state time-update matrix
218  # - "Q": The process noise matrix
