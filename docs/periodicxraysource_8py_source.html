403  ## @fun #getSignal is responsible for returning the photon flux from the
404  # pulsar at a given time
405  #
406  # @details The #getSignal method is the method which is responsible for
407  # returning the current value of flux from the signal source at a given
408  # time. If there is uncertainty in the time, then the expected value of
409  # the signal is returned. Uncertainty in time is indicated by the
410  # optional tVar argument.
411  #
412  # In every case, the method calls the #getPhase method to determine the
413  # current signal source phase. If no uncertainty in time is passed, then
414  # the #getPulseFromPhase method is called to lookup/interpolate the
415  # current flux based on the phase and the signal #profile.
416  #
417  # If a value is passed for tVar, then the process is more complicated.
418  # Rather than simply look up the signal value from the phase, the function
419  # returns an approximation of the expected value of the signal, given the
420  # mean and variance of time.
421  #
422  # Normally, the expected value would be computed by integrating the
423  # product of the true distribution of time (probably a normal distribution)
424  # with the flux as a function of time. However, due to the non-analytical
425  # nature of the flux function, the direct computation of this integral is
426  # intractable. To overcome this limitation, the time distributuion is
427  # approximated as a moment-matched uniform distribution. Using this
428  # approximation, the approximate integral may be directly computed simply
429  # by looking up the start and end values of #singlePeriodIntegral, and
430  # dividing by the appropriate scaling factor.
431  #
432  # @param self The object pointer
433  # @param observatoryTime The time for which to compute the signal
434  # @param tVar (optional) The variance of the time estimate
