313  ## @fun #getSignal is responsible for returning the photon flux from the
314  # pulsar at a given time
315  #
316  # @details The #getSignal method is the method which is responsible for
317  # returning the current value of flux from the signal source at a given
318  # time. If there is uncertainty in the time, then the expected value of
319  # the signal is returned. Uncertainty in time is indicated by the
320  # optional tVar argument.
321  #
322  # In every case, the method calls the #getPhase method to determine the
323  # current signal source phase. If no uncertainty in time is passed, then
324  # the #getPulseFromPhase method is called to lookup/interpolate the
325  # current flux based on the phase and the signal #profile.
326  #
327  # If a value is passed for tVar, then the process is more complicated.
328  # Rather than simply look up the signal value from the phase, the function
329  # returns an approximation of the expected value of the signal, given the
330  # mean and variance of time.
331  #
332  # Normally, the expected value would be computed by integrating the
333  # product of the true distribution of time (probably a normal distribution)
334  # with the flux as a function of time. However, due to the non-analytical
335  # nature of the flux function, the direct computation of this integral is
336  # intractable. To overcome this limitation, the time distributuion is
337  # approximated as a moment-matched uniform distribution. Using this
338  # approximation, the approximate integral may be directly computed simply
339  # by looking up the start and end values of #singlePeriodIntegral, and
340  # dividing by the appropriate scaling factor.
341  #
342  # @param self The object pointer
343  # @param observatoryTime The time for which to compute the signal
344  # @param tVar (optional) The variance of the time estimate
