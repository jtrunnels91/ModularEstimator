10 ## @class SubState
11 # @brief This is an abstract base class for objects used as sub-states in
12 # State.ModularFilter.
13 #
14 # @details SubState is an abstract base class that specifies the methods which
15 # are required for an object to function as a sub-state of State.ModularFilter.
16 #
17 # Some of these methods are implemented and most likely do not need to be
18 # reimplemented in a derived class implementation (for example the #dimension
19 # and #covariance methods.
20 #
21 # Other methods may have a rudimentary implementation that may be suitable for
22 # some derived classes, but not others, depending on the specific
23 # functionality of the derived class (for instance #getStateVector and
24 # #storeStateVector).
25 #
26 # Finally, some methods are specifically tagged as abstract methods and are
27 # not implemented at all. These methods must be implemented in the derived
28 # class. This is usually because there is no way to implement even a
29 # rudimentary version of what the method is supposed to do without having some
30 # knowledge of what kind of substate the derived class contains (for instance
31 # #timeUpdate and #getMeasurementMatrices).
32 #
33 # In any case, the documentation for each method of SubState contains a
34 # generalized description of what functionality the implementation should
35 # provide in a derived class.
36 class SubState():

147  # @details

214  ## @fun #timeUpdate returns time-update matrices
215  #
216  # @details The #timeUpdate method is responsible for returning the EKF
217  # time update measurement matrices. Specifically, it returns the state
218  # update matrix \f$\mathbf{F}\f$ and the process noise matrix
219  # \f$\mathbf{Q}\f$, following the standard
220  # <a href="https://en.wikipedia.org/wiki/Extended_Kalman_filter">
221  # Extended Kalman Filter</a> time update equations:
222  #
223  # \f[\sv[timeIndex=k+1, aPriori=True] =
224  # \mathbf{F} \sv[timeIndex=k, aPriori=False] \f]
225  # \f[\svVar[timeIndex=k+1, aPriori=True] =
226  # \mathbf{F} \svVar[timeIndex=k+1, aPriori=True] \mathbf{F}^{T} + \mathbf{Q} \f]
227  #
228  # Because these matrices are nescessarily specific to the type of substate
229  # being updated, there is no default implementation in the SubState class.
230  # Rather, each derived class must implement this method as appropriate for
231  # the dynamics of the state being modeled.
232  #
233  # In addition, some substates may require additional operations to occur
234  # at a time update. For instance, if a substate includes auxillary values
235  # (for instance, the attitude quaternion derived from the attitude error
236  # state), it may need to be time-updated seperately from the other states.
237  # In this case, the local implementation of the #timeUpdate function is
238  # the place to do these updates.
239  #
240  # @param self The object pointer
241  # @param dT The ellapsed time over which the time update occurs
242  # @param dynamics A dictionary containing any dynamics infomation which
243  # may be needed to update the state, for instance, measured accelerations
244  # or angular velocities.
245  #
246  # @returns A dictionary containing, at minimum, the following items:
247  # - "F": The state time-update matrix
248  # - "Q": The process noise matrix
249  @abstractmethod
250  def timeUpdate(self, dT, dynamics=None):
251  pass
