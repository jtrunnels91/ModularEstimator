11 ## @class SubState
12 # @brief This is an abstract base class for objects used as sub-states in
13 # State.ModularFilter.
14 #
15 # @details SubState is an abstract base class that specifies the methods which
16 # are required for an object to function as a sub-state of State.ModularFilter.
17 #
18 # Some of these methods are implemented and most likely do not need to be
19 # reimplemented in a derived class implementation (for example the #dimension
20 # and #covariance methods.
21 #
22 # Other methods may have a rudimentary implementation that may be suitable for
23 # some derived classes, but not others, depending on the specific
24 # functionality of the derived class (for instance #getStateVector and
25 # #storeStateVector).
26 #
27 # Finally, some methods are specifically tagged as abstract methods and are
28 # not implemented at all. These methods must be implemented in the derived
29 # class. This is usually because there is no way to implement even a
30 # rudimentary version of what the method is supposed to do without having some
31 # knowledge of what kind of substate the derived class contains (for instance
32 # #timeUpdate and #getMeasurementMatrices).
33 #
34 # In any case, the documentation for each method of SubState contains a
35 # generalized description of what functionality the implementation should
36 # provide in a derived class.
37 class SubState():

197  ## @fun #timeUpdate returns time-update matrices
198  #
199  # @details The #timeUpdate method is responsible for returning the EKF
200  # time update measurement matrices. Specifically, it returns the state
201  # update matrix \f$\mathbf{F}\f$ and the process noise matrix
202  # \f$\mathbf{Q}\f$, following the standard
203  # <a href="https://en.wikipedia.org/wiki/Extended_Kalman_filter">
204  # Extended Kalman Filter</a> time update equations:
205  #
206  # \f[\sv[timeIndex=k+1, aPriori=True] =
207  # \mathbf{F} \sv[timeIndex=k, aPriori=False] \f]
208  # \f[\svVar[timeIndex=k+1, aPriori=True] =
209  # \mathbf{F} \svVar[timeIndex=k+1, aPriori=True] \mathbf{F}^{T} + \mathbf{Q} \f]
210  #
211  # Because these matrices are nescessarily specific to the type of substate
212  # being updated, there is no default implementation in the SubState class.
213  # Rather, each derived class must implement this method as appropriate for
214  # the dynamics of the state being modeled.
215  #
216  # In addition, some substates may require additional operations to occur
217  # at a time update. For instance, if a substate includes auxillary values
218  # (for instance, the attitude quaternion derived from the attitude error
219  # state), it may need to be time-updated seperately from the other states.
220  # In this case, the local implementation of the #timeUpdate function is
221  # the place to do these updates.
222  #
223  # @param self The object pointer
224  # @param dT The ellapsed time over which the time update occurs
225  # @param dynamics A dictionary containing any dynamics infomation which
226  # may be needed to update the state, for instance, measured accelerations
227  # or angular velocities.
228  #
229  # @returns A dictionary containing, at minimum, the following items:
230  # - "F": The state time-update matrix
231  # - "Q": The process noise matrix
232  @abstractmethod
233  def timeUpdate(self, dT, dynamics=None):
234  pass
